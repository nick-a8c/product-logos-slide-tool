# v2.2 — Guided Setup, Templates & Tutorial

Onboarding + settings-access redesign. Goal: make the tool easy to enter and configure for any user (marketer, designer, sales, dev) — clear steps, easy to find things. Settled with the owner (Nebojsa); this doc is the build contract.

## Decisions (locked)

- **Wizard axis:** role → use-case (role is a light touch that pre-highlights use-cases; use-case drives the real settings).
- **Wizard entry:** dismissible modal *over* the tool (tool loads with defaults behind; closing = "I'll set it up myself"). Same component powers first-run and the reopen ("recall") button.
- **Tutorial:** inline animated demos as **Lottie JSON** (owner authors the clips). A Lottie player is inlined (file-size hit accepted). Fully offline.
- **Scope:** all three features ship in 2.2 — wizard + recall modal + first-visit tutorial + persistent "How it works" button.
- **Templates:** new **in-app named templates** in `localStorage` (separate from the existing JSON preset export/import, which stays).
- **"Brands," not teams:** the picker chooses which of the tool's built-in brand logos appear. There are no teams. Source = `DEFAULT_ICONS` (`'01'`..`'25'`).

## First-run entry — unified Welcome fork

One small **Welcome** modal (no stacked modals) with three choices:
- **Guided setup** → opens the wizard
- **Show me how it works** → opens the tutorial
- **Just start** → dismiss to the default tool

Returning visitors see nothing; wizard + tutorial remain reachable from persistent buttons. Gated by `state.onboarded`.

## The wizard

Dismissible modal (X / Esc / click-outside / "Skip — I'll set it up myself"). Steps:

| Step | User picks | Notes |
|---|---|---|
| 0 · (optional) Template | Start from a saved template | Pick-list only shows if templates exist |
| 1 · Role | Marketing · Design · Sales · Engineering · Just exploring | Only pre-sorts/pre-highlights use-cases + tailors wording. Sets nothing hard. |
| 2 · Use-case | cards (pre-sorted by role) | **Drives real settings** via the mapping table |
| 3 · Quick tweaks | Brands, Background, Size tier | The few things worth asking; rest auto-derived |
| 4 · Review | "Open in tool" · "Save as template" | Writes `state`, runs apply/render, closes |

**Wizard-exposed** choices: role, use-case, brand set, background, size tier.
**Auto-derived** from use-case: layout, ratio, format, "feel," resolution. Everything stays fully editable in the tool afterward.

## Use-case → settings mapping

| Use-case | Layout | Ratio | Format | Size tier | Background | Feel |
|---|---|---|---|---|---|---|
| Social Reel / Story | Line | 9:16 | MP4 | Standard | Solid brand | Snappy |
| Social post (square) | Line | 1:1 | MP4 | Standard | Solid brand | Balanced |
| Slide / Presentation | Line | 16:9 | MP4 | Large | Solid | Balanced |
| Website hero | Ring | 21:9 | Interactive HTML | Large | Gradient | Calm |
| Logo loop / Sticker | Ring | 1:1 | WebM (alpha) | Standard | Off (transparent) | Calm |
| Email / Banner | Line | 16:9 | PNG still | Standard | Solid | Balanced |
| Wallpaper / Zoom bg | Line | 16:9 | MP4 | Large | Gradient | Calm |

**Role → suggested use-cases** (pre-highlight only):
- Marketing → Social Reel, Social square, Email/Banner
- Design → Website hero, Logo loop, Slide
- Sales → Slide, Email/Banner, Wallpaper
- Engineering → Website hero, Logo loop
- Just exploring → all shown

**"Feel"** = a small preset over the real params: Calm (slow / small slide), Balanced (current defaults), Snappy (fast / bigger stagger). In Ring it maps to speed/tilt.

## Templates

- New key `icon-stage-templates-v1` = `[{ id, name, createdAt, state }]`, independent of main state key `icon-stage-v2`.
- "Save as template" at wizard end **and** a small panel button. Templates surface as step-0 pick-list in the wizard.
- Existing JSON preset export/import (`..._preset_..._v2.json`) stays for cross-machine sharing.

## Tutorial

- First-run (via Welcome fork) + persistent **bright "How it works" button** (bottom corner) to reopen anytime.
- Modal: Next / Back / Skip + progress dots; **one Lottie clip per step**.
- Draft steps (5 clips, owner-authored): ① pick brands → ② Line vs Ring → ③ tune animation/timeline → ④ background & aspect → ⑤ export.
- Build the shell with placeholder clips; owner drops in real Lottie JSON later.

## State / migration

- Add `state.onboarded`, `state.tutorialSeen` (default `false`).
- **Migration in `restore()`:** for anyone with existing saved state, set both **`true`** so current users aren't re-onboarded. New visitors (no saved state) get `false`.
- Templates key is additive (no migration).

## Implementation notes (real hooks in `index.html`)

- Reuse existing modal pattern: `.modal-bg` / `.modal` / `.modal-actions` (used by `openPicker`, `#modalBg`).
- Layout: `state.layout` (`'line'|'ring'`), `data-layout` on `.app`; toggle logic near `setLayout` (~L7568), buttons `#btn-layout-line` / `#btn-layout-ring`.
- Aspect: `applyAspectRatio(ar)` (~L4746) / `morphToAspect(ar)`. Ratios in `ASPECT_RATIOS`.
- Size tier: `populateResolutionDropdown()` (~L4531) builds Small/Standard/Large from the aspect's base dims.
- Background: `state.bgOn` / `state.bgMode` (`solid|linear|radial`) / `state.bgColor` / `bgColorB` etc.; buttons `#btn-bgmode-*` (~L7745); one source of truth `getBgCSS()` / `fillBgCanvas()`.
- Brands: `DEFAULT_ICONS` (`'01'`..`'25'`), `state.library`, `state.assignments` (slot→id), `ensureAssignments()`, `DEFAULT_LAYOUTS`.
- Format + quality: `state.exportRange`, `state.videoQuality` (`web|high|max`).
- Apply pipeline: write `state` → `bindUIFromState()` → `renderStage()` → `applyZoom()` → `persist()`.

## Build sequence

1. **Foundation** — state flags + migration + Welcome fork modal + persistent buttons (Guided setup / How it works). *Verify: first-run shows Welcome; returning visitor doesn't; buttons reopen.*
2. **Wizard** — role → use-case → tweaks → review; mapping engine writes state and applies. *Verify: each use-case lands the mapped settings in the live tool.*
3. **Recall** — panel button reopens the same wizard. *Verify: reopen mid-session applies cleanly.*
4. **Templates** — localStorage store + save/load UI + wizard step-0 pick-list. *Verify: save, reload page, template restores.*
5. **Tutorial** — modal shell + inlined Lottie player + persistent button + placeholder clips. *Verify: steps navigate; reopen works; reduced-motion parks.* (Real clips dropped in when owner delivers Lottie JSON.)

Keep `index.html` and `product-logos-slide-tool.html` byte-identical after each phase. Parse-check script blocks; verify in preview.
